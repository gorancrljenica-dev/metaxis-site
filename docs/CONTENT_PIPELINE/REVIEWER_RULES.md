# Reviewer Rules

Reviewer is a human (or evaluator agent) who checks drafts before approval.
These rules are the minimum bar. Below the bar = reject or rewrite.

---

## Reject If

**Tone failures:**
- Sounds like a startup guru
- Motivational without operational grounding
- Uses urgency bait ("you must", "don't miss", "now more than ever")
- Abstract claims without specifics ("AI is transforming everything")
- Fake authority ("as an expert in...")
- "Future of AI / work / business" framing without a concrete point

**Language failures:**
- Generic AI phrasing ("leverage AI", "harness the power of", "game-changer")
- Corporate fluff ("synergy", "holistic approach", "value-add")
- Engagement bait ("comment below", "what do you think?", "share if you agree")
- Emoji spam (more than 2 functional emojis)
- AI-slop hooks ("In today's fast-paced world...", "As we navigate...")

**Structural failures:**
- Platform mismatch (LinkedIn copy on Instagram, IG caption format on LinkedIn)
- Content that does not trace back to the source blog post
- Claims that cannot be verified from the source material

---

## Approve If

- Specific operational insight (not vague advice)
- Tone is calm, direct, human
- Platform format is correct
- Content is traceable to source material
- No rejected patterns present
- Passes a quick "would I share this?" test from the Metaxis perspective

---

## Reviewer Priority Order

1. **Specificity** — Is there a concrete, real point being made?
2. **Platform fit** — Does the format match where it will be posted?
3. **Tone** — Does it sound like Metaxis, not like a content mill?
4. **Traceability** — Can you find the source claim in the blog post?
5. **No slop** — Does it pass REJECTED_PATTERNS.md?

---

## Reviewer Is Not

- An editor who rewrites from scratch
- A creativity judge
- Responsible for grammar alone

Reviewer approves, rejects, or requests targeted edits. Not wholesale rewrites.
