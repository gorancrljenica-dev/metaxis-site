# Sprint 1 V2 Review — blog-001

---

## LinkedIn — v1 vs v2

**Changes applied:**
- Final sentence moved to opening. Post restructured: claim → definition → loop → phases → stop condition → implication.
- "actually" removed from phase intro.
- "without losing the thread halfway through" removed.

**Did it improve?**
Structurally yes. Opening with the strongest claim ("Most AI automations... aren't missing a better model. They're missing control flow.") is more useful to the reader than opening with the abstract distinction. The reader knows immediately what the post is about and why it matters.

**New problem introduced:**
The ending is now weaker. V1 ended on the strongest line. V2 ends on: "An AI that answers a question and an AI that completes a task are different systems. The loop is what makes the second one work." This is a reasonable closing sentence, but it's slightly softer than what it replaced. Moving the best line to the opener gains a sharper entry but trades away the payoff ending.

**What still sounds AI-generated:**
- The bold **Think** / **Act** / **Verify** phase structure. Functional, traceable to source, but the format pattern (bold label — short description) is common in AI-generated LinkedIn content. It's not slop, but it reads as template.
- "Both outcomes look identical from the outside." — slightly over-clean phrasing. Real operators would say "you can't tell until you check the output."

**Verdict:** Pass. Better entry, weaker exit compared to V1. Neither version is clearly superior — they optimize for different things.
**V1 was better at:** strong, specific close.
**V2 is better at:** immediate clarity on the central claim.
**Recommendation:** Test both. The right choice depends on how LinkedIn's algorithm weighs opening vs. closing engagement. As a writing quality call: V2 opener + V1 closing line as final sentence.

## Instagram — v1 vs v2

**Changes applied:**
- Closing paragraph cut entirely.
- Phase description fixed: "check the result before moving forward" → "did it work? If not, return to Think."
- Visual direction note separated into a clearly marked production note below the copy.

**Did it improve?**
Yes, cleanly. The V1 closing paragraph was the only genuine weakness and it's gone. The V2 ends on "The model isn't the problem. The control flow is." — that's a clean, specific close. No summarizing. No landing statement.

The fixed phase description is also tighter. "Did it work? If not, return to Think." is more operational and more faithful to the source than "check the result before moving forward."

**What still sounds AI-generated:**
- The opening contrast ("An AI that answers is a tool. An AI that loops is a system.") — this is the strongest line in the caption and genuinely good, but the two-sentence parallel structure is a pattern AI writers default to. It works here, but future posts should vary the entry structure to avoid a recognizable template.
- The three-phase breakdown is inherently list-structured. For Instagram, it's appropriate (visual scanning), but it means the caption has a predictable internal rhythm.

**Verdict:** Pass without conditions. This is the clearest improvement in the V2 cycle. Ready for human approval.

## Facebook — v1 vs v2

**Changes applied:**
- Opening sentence tightened ("starts behaving strangely over multiple steps" → "breaks down over multiple steps").
- "This isn't about building complex agents." removed.
- Closing paragraph restructured: the quotable distinction now functions as a bridge sentence before the blog link, rather than a standalone defensive paragraph.

**Did it improve?**
Yes, but modestly. Removing the defensive sentence is the meaningful change. The post reads with more confidence now. The bridging sentence ("'I set up an AI automation' and 'I have a reliable AI automation' are not the same statement. The gap is usually control flow.") works well — it lands the key insight and then the blog link feels natural rather than abrupt.

**What still sounds AI-generated:**
- "AI systems that handle multi-step tasks need a loop structure: Think (plan the next move), Act (execute one thing), Verify (check the result before continuing)." — the parenthetical inline definitions are clean but slightly over-explained. A human writer who knows the audience would probably trust them to follow Think/Act/Verify without the parenthetical glosses.
- The rhythm of the post is still very structured: scenario → problem diagnosis → framework → missing element → quotable line → blog link. It works, but it's a recognizable formula.

**Verdict:** Pass. Stronger than V1. The parenthetical definitions are a minor issue, not a rejection trigger.

## Remaining AI patterns across all V2 drafts

1. **Parallel two-sentence openers** (Instagram) — "X is A. Y is B." works, but becomes recognizable as a template if repeated.
2. **Bold phase labels as scannable structure** (LinkedIn, Instagram) — functional, but common in AI-generated educational content. Should vary in future posts.
3. **Over-explained inline definitions** (Facebook) — parenthetical glosses on terms that the audience likely knows.
4. **Post rhythm predictability** — all three posts follow: observation → diagnosis → framework → insight → close. The formula works for this topic. It will wear thin if all Metaxis posts follow the same arc.
