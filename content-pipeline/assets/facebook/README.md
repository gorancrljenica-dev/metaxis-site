# Facebook Visual Assets

## What works

- **Operational screenshots (Category B):** Real workflow, terminal output, tool UI. Grounding for a scenario-based post.
- **Simple typography (Category A):** A single quotable line from the post. Works when the post has a strong quotable claim.
- **No image:** Facebook text posts are viable and sometimes outperform posts with generic images. Use no image if the right visual isn't available.
- **Link preview:** If the post links to a blog post, the link preview card is the visual. Ensure metaxis.io og:image tags are set correctly.

## Aspect ratios

- `1200 × 630` (1.91:1) — matches Facebook link preview dimensions; use this to maintain visual consistency between image posts and link previews
- `1080 × 1080` (1:1) — acceptable for standalone image posts

## Pairing logic

Facebook captions are mid-length and scenario-based. The visual supports the scenario, not the framework.

- Post about a broken workflow → screenshot of a real workflow or log output
- Post with a quotable distinction → Category A card with the distinction in type
- Post linking to a blog → let the link preview serve as the visual; ensure the blog og:image is correct

Do not use a complex diagram on Facebook — the audience skews less technical than LinkedIn/IG; a diagram without explanation loses them.

## What to avoid

- Instagram carousel (wrong format for Facebook feed)
- Visuals that require reading to understand (Facebook users read captions, not images)
- Anything that looks like a motivational poster
- Stock imagery
